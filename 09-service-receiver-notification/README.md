<h1><a name="_8cuic5pbpza"></a><span lang=ru>9. Service, BroadcastReciever</span></h1>

<h2 style='margin-top:0cm;margin-right:0cm;margin-bottom:15.0pt;margin-left:
0cm;line-height:117%;mso-pagination:widow-orphan;page-break-after:auto'><a
name="_fsp0esl6dboy"></a><span lang=ru>Broadcasts overview</span></h2>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121'>Android apps can send or receive broadcast messages from the
Android system and other Android apps, similar to the </span><u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:

broadcasts are sent when an event of interest occurs. For example, the Android
system sends broadcasts when various system events occur, such as when the
system boots up or the device starts charging. Apps can also send custom
broadcasts, for example, to notify other apps of something that they might be
interested in (for example, some new data has been downloaded).<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:

broadcast is sent, the system automatically routes broadcasts to apps that have
subscribed to receive that particular type of broadcast.<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121'>Generally speaking, broadcasts can be used as a messaging system
across apps and outside of the normal user flow. However, you must be careful
not to abuse the opportunity to respond to broadcasts and run jobs in the
background that can contribute to a slow system performance, as described in
the following video.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=ru style='font-size:12.0pt;line-height:115%;

<h2><a name="_xj4lvnywk7eo"></a><span lang=ru>Сервис</span></h2>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><u><span lang=ru style='font-family:"Roboto Mono";
mso-fareast-font-family:"Roboto Mono";mso-bidi-font-family:"Roboto Mono";
color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/app/Service.html"><span
style='color:#039BE5'>Service</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'> является компонентом
приложения, который может выполнять длительные операции в фоновом режиме и не
содержит пользовательского интерфейса. Другой компонент приложения может
запустить службу, которая продолжит работу в фоновом режиме даже в том случае,
когда пользователь перейдет в другое приложение. Кроме того, компонент может
привязаться к службе для взаимодействия с ней и даже выполнять межпроцессное
взаимодействие (IPC). Например, служба может обрабатывать сетевые транзакции,
воспроизводить музыку, выполнять ввод-вывод файла или взаимодействовать с
поставщиком контента, и все это в фоновом режиме.<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121'>Фактически служба может принимать две формы:<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm;line-height:150%'><b style='mso-bidi-font-weight:normal'><span
lang=ru style='font-size:12.0pt;line-height:150%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'>Запущенная<o:p></o:p></span></b></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121'>Служба является «запущенной», когда компонент приложения
(например, операция) запускает ее вызовом </span><u><span lang=ru
style='font-family:"Roboto Mono";mso-fareast-font-family:"Roboto Mono";
mso-bidi-font-family:"Roboto Mono";color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/content/Context.html#startService(android.content.Intent)"><span
style='color:#039BE5'>startService()</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'>. После запуска служба может
работать в фоновом режиме в течение неограниченного времени, даже если
уничтожен компонент, который ее запустил. Обычно запущенная служба выполняет
одну операцию и не возвращает результатов вызывающему компоненту. Например, она
может загружать или выгружать файл по сети. Когда операция выполнена, служба
должна остановиться самостоятельно.<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm;line-height:150%'><b style='mso-bidi-font-weight:normal'><span
lang=ru style='font-size:12.0pt;line-height:150%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'>Привязанная<o:p></o:p></span></b></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121'>Служба является «привязанной», когда компонент приложения
привязывается к ней вызовом </span><u><span lang=ru style='font-family:"Roboto Mono";
mso-fareast-font-family:"Roboto Mono";mso-bidi-font-family:"Roboto Mono";
color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/content/Context.html#bindService(android.content.Intent,%20android.content.ServiceConnection,%20int)"><span
style='color:#039BE5'>bindService()</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'>. Привязанная служба
предлагает интерфейс клиент-сервер, который позволяет компонентам
взаимодействовать со службой, отправлять запросы, получать результаты и даже
делать это между разными процессами посредством межпроцессного взаимодействия
(IPC). Привязанная служба работает только пока к ней привязан другой компонент
приложения. К службе могут быть привязаны несколько компонентов одновременно,
но когда все они отменяют привязку, служба уничтожается.<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121'>Хотя в этой документации эти два типа служб обсуждаются
отдельно, служба может работать обеими способами — она может быть запущенной (и
работать в течение неограниченного времени) и допускать привязку. Это зависит
от реализации пары методов обратного вызова: </span><u><span lang=ru
style='font-family:"Roboto Mono";mso-fareast-font-family:"Roboto Mono";
mso-bidi-font-family:"Roboto Mono";color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/app/Service.html#onStartCommand(android.content.Intent,%20int,%20int)"><span
style='color:#039BE5'>onStartCommand()</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'> позволяет компонентам
запускать службу, а </span><u><span lang=ru style='font-family:"Roboto Mono";
mso-fareast-font-family:"Roboto Mono";mso-bidi-font-family:"Roboto Mono";
color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/app/Service.html#onBind(android.content.Intent)"><span
style='color:#039BE5'>onBind()</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'> позволяет выполнять
привязку.<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121'>Независимо от состояния приложения (запущенное, привязанное или
и оба сразу) любой компонент приложения может использовать службу (даже из
отдельного приложения) подобно тому, как любой компонент может использовать
операцию — запустив ее с помощью </span><u><span lang=ru style='font-family:
"Roboto Mono";mso-fareast-font-family:"Roboto Mono";mso-bidi-font-family:"Roboto Mono";
color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/content/Intent.html"><span
style='color:#039BE5'>Intent</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'>. Однако вы можете объявить
закрытую службу в файле манифеста и заблокировать доступ к ней из других
приложений. Более подробно это обсуждается в разделе </span><u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#039BE5'><a
href="https://developer.android.com/guide/components/services#Declaring"><span
style='color:#039BE5'>Объявление службы в манифесте</span></a></span></u><span
lang=ru style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121'>.<o:p></o:p></span></p>

<p class=MsoNormal><b style='mso-bidi-font-weight:normal'><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121;background:#F7F7F7'>Служба —
это просто компонент, который может выполняться в фоновом режиме, даже когда
пользователь не взаимодействует с приложением. Следовательно, вы должны
создавать службу только в том случае, если вам нужно именно это.<o:p></o:p></span></b></p>

<p class=MsoNormal><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121;background:#F7F7F7'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121;background:#F7F7F7'>Помните, что если вы действительно
используете службу, она выполняется в <b style='mso-bidi-font-weight:normal'>основном
потоке вашего приложения</b> по умолчанию, поэтому вы должны создать новый
поток в службе, если она выполняет интенсивные или блокирующие операции.<o:p></o:p></span></p>

<p class=MsoNormal><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121;background:#F7F7F7'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121;background:#F7F7F7'>Чтобы создать службу, необходимо создать
подкласс класса </span><u><span lang=ru style='font-family:"Roboto Mono";
mso-fareast-font-family:"Roboto Mono";mso-bidi-font-family:"Roboto Mono";
color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/app/Service.html"><span
style='color:#039BE5'>Service</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121;background:#F7F7F7'> (или
одного из существующих его подклассов). В вашей реализации необходимо
переопределить некоторые методы обратного вызова, которые обрабатывают ключевые
моменты жизненного цикла службы и при необходимости предоставляют механизм
привязывания компонентов. Наиболее важные методы обратного вызова, которые
необходимо переопределить:<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm;line-height:150%'><span lang=ru><a
href="https://developer.android.com/reference/android/app/Service.html#onStartCommand(android.content.Intent,%20int,%20int)"><span

mso-bidi-font-family:"Roboto Mono";color:#039BE5;background:#F7F7F7'>onStartCommand()<o:p></o:p></span></a></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;
font-family:Roboto;mso-fareast-font-family:Roboto;mso-bidi-font-family:Roboto;
color:#212121;background:#F7F7F7'>Система вызывает этот метод, когда другой
компонент, например, операция, запрашивает запуск этой службы, вызывая </span><u><span
lang=ru style='font-family:"Roboto Mono";mso-fareast-font-family:"Roboto Mono";
mso-bidi-font-family:"Roboto Mono";color:#039BE5;background:#F7F7F7'><a

style='color:#039BE5'>startService()</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121;background:#F7F7F7'>. После
выполнения этого метода служба запускается и может в течение неограниченного
времени работать в фоновом режиме. Если вы реализуете такой метод, вы обязаны
остановить службу посредством вызова </span><u><span lang=ru style='font-family:
"Roboto Mono";mso-fareast-font-family:"Roboto Mono";mso-bidi-font-family:"Roboto Mono";
color:#039BE5;background:#F7F7F7'><a
href="https://developer.android.com/reference/android/app/Service.html#stopSelf()"><span
style='color:#039BE5'>stopSelf()</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:

lang=ru style='font-family:"Roboto Mono";mso-fareast-font-family:"Roboto Mono";

style='color:#039BE5'>stopService()</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121;background:#F7F7F7'>. (Если
требуется только обеспечить привязку, реализовывать этот метод не обязательно).<o:p></o:p></span></p>

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:

<p class=MsoNormal style='margin-top:12.0pt;margin-right:0cm;margin-bottom:
12.0pt;margin-left:0cm'><span lang=ru style='font-size:12.0pt;line-height:115%;

color:#212121;background:#F7F7F7'>Система вызывает этот метод, когда другой

реализации этого метода вы должны обеспечить интерфейс, который клиенты
используют для взаимодействия со службой, возвращая </span><u><span lang=ru
style='font-family:"Roboto Mono";mso-fareast-font-family:"Roboto Mono";
mso-bidi-font-family:"Roboto Mono";color:#039BE5;background:#F7F7F7'><a

style='color:#039BE5'>IBinder</span></a></span></u><span lang=ru
style='font-size:12.0pt;line-height:115%;font-family:Roboto;mso-fareast-font-family:
Roboto;mso-bidi-font-family:Roboto;color:#212121;background:#F7F7F7'>. Всегда
необходимо реализовывать этот метод, но если вы не хотите разрешать привязку,
необходимо возвращать значение null.<o:p></o:p></span></p>

12.0pt;margin-left:0cm;line-height:150%'><span lang=ru><a

служба уже запущена, этот метод не вызывается.<o:p></o:p></span></p>

     активити на котором будет отображаться погода. </span></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo2'><span lang=ru>При старте
     активити должен запускаться Service через bind service. Т.е. сервис должен
     автоматически уничтожаться, когда уничтожается активити.</span></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo2'><span lang=ru>Каждую
     минуту сервис должен ходить по адресу и получать текущую температуру <u><span
     style='color:#1155CC'><a
     href="http://api.openweathermap.org/data/2.5/weather?q=saransk&amp;units=metric&amp;appid=a924f0f5b30839d1ecb95f0a6416a0c2"><span
     style='color:#1155CC'>http://api.openweathermap.org/data/2.5/weather?q=saransk&amp;units=metric&amp;appid=a924f0f5b30839d1ecb95f0a6416a0c2</span></a></span></u>
     <br>
     Результат о полученной погоде нужно передавать в активити, не используя
     широковещательные сообщения. Нужно делать, как описывается в этом посте: <u><span
     style='color:#1155CC'><a
     href="https://stackoverflow.com/a/23587641/1120982"><span
     style='color:#1155CC'>https://stackoverflow.com/a/23587641/1120982</span></a></span></u>
     Т.е. в активити должна быть вью, которая обновляется после того, как
     сервис получил погоду</span></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo2'><span lang=ru>На активити
     добавить кнопку - скачать файл. При нажатии на кнопку должен запускаться
     сервис через startService() При запуске сервиса в него нужно передавать
     ссылку на .zip архив с картинкой. Сформировать архив нужно самостоятельно.
     Взять любую картинку, желательно размером побольше, зазиповать ее, и
     выложить в удобное для вас место, чтобы была прямая ссылка на файл. </span></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo2'><span lang=ru>Если закрыть
     активити, то сервис не должен завершаться. Сервис должен завершаться
     тогда, когда файл будет скачан и распакован. </span></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo2'><span lang=ru>Пометить
     сервис, как foreground сервис. </span></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo2'><span lang=ru>Алгоритм
     работы сервиса: при старте он получает в себя ссылку на архив, начинает
     скачиваться во внутреннюю память (чтобы не иметь дело с разрешениями) Во
     время скачки прогресс скачивания нужно отправлять в активити через
     широковещательные сообщения. Как только скачка завершилась, нужно, чтобы
     файл распаковался. После этого передать путь к распакованной картинке в
     активити<span style='mso-spacerun:yes'>  </span>и там отобразить ее в
     imageView. После этого сервис должен завершиться.</span></li>
</ol>
